How Do CPUs Work?

While CPUs may seem daunting as a whole, they are quite simple once broken down into simpler sections. Through this guide, you will get a fundamental understanding of registers, busses, the ALU or Arithmetic Logic Unit, RAM or Random Access Memory (which is not in a CPU), program counters, and the control unit. With this, you will have a basic understanding of how the components of a computer work together to complete basic tasks.

A note for this guide. The following information will be aligned with the 8-bit CPU built and viewed in the stream. It will be noted whenever a piece information is specific to the 8-bit CPU. In addition, this guide will be a brief overview. To get more information, we suggest you do some more research.

Address and Data Bus

Busses are ways that information can be moved around within the CPU. Most CPUs have an address and a data bus, but in the 8-bit CPU on this site has a shared bus.

Starting with the address bus, this is how the CPU accesses different words (8 bits) of data. Because the data is limited by size, we must have a way to reference different data at different times. We can use addresses in conjunction with memory to store data at different “locations” that can be later accessed. To retrieve or set a specific set of data, the CPU can use the address bus to specify the destination address of the data (pictured right). For example, at location 4010, the word “0000 0100” is present.

Next is the data bus, this is how the CPU moves the data around itself. This is done by one module outputting it’s data on the bus, and another module “taking in” the data.

In most cases, the address and data busses take up separate “wires” on the system bus. But, on the 8-bit CPU on this site, the main bus is shared between the two. This is done by changing how the microcode runs on the instructions (more about that later).

Arithmetic Logic Unit (ALU)

The Arithmetic Logic Unit or ALU is responsible for taking one or two operands and performing a arithmetic or logical operation. It then will return the result of the operation. Operations that can be performed often include addition, subtraction, OR, XOR, AND, and NOT, thought this can vary between CPU’s. In the case of the 8-bit CPU on this site, it can perform addition and subtraction.

Registers

Registers are a form of memory that can be accessed faster than RAM. They are loaded by the machine in order to complete arithmetic operations with the ALU (as well as some other tasks). Typically, there is are two registers for input on the ALU, and one as output. Many CPU’s include more. In the case of the 8-bit CPU, there are two input registers (A and B) for the ALU, as well as one output register for the CPU. In addition, there is one instruction register for holding the current instruction, program counter register, and a “flags” or status register (to be used with jumps).

Program Counter

The program counter is a register that indicates where a computer is in program sequence. After fetching an instruction, it is incremented and then points to the location of the next instruction.

Control Unit

The program counter is a register that indicates where a computer is in program sequence. After fetching an instruction, it is incremented and then points to the location of the next instruction.

Random Access Memory (RAM)

Random Access Memory is a component of a computer not within the CPU. As another form of computer memory, it is often tasked with storing data and instructions for the CPU. For the 8-bit CPU, it is included so programs can be created.
